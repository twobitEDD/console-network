# The Console Network Registry

> How themes and projects get from your computer into every running console in the network.

The registry is a plain-text, file-based directory living at the root of the [`console-network`](https://github.com/twobitEDD/console-network) monorepo under `registry/`. Everything you see in the built-in **Connections** panel is in there. Everything that ships with `@twobitedd/console-network` as a "theme" is in there. All of it is publicly editable via pull requests.

Every entry is a folder. The folder name is the entry's `id` and `slug` тАФ the validator will reject mismatches.

## How changes propagate

1. You open a PR that adds/edits a folder under `registry/`.
2. CI runs `node scripts/validate-registry.mjs` and `node scripts/build-registry.mjs --check`. Green = safe to review.
3. A maintainer sanity-checks the PR and merges.
4. On merge, `registry/index.json` is served from `main` at:

   ```
   https://raw.githubusercontent.com/twobitEDD/console-network/main/registry/index.json
   ```

5. Every running console that mounts `<ConsoleHost connections={тАж} />` fetches that URL the first time a user opens the Connections panel. jsdelivr and GitHub's raw CDN cache it for a few minutes; after that your entry appears.

## Federation

The registry is opinionated but not authoritative. If you want to run your own mirror or your own curated list, point your hosts at a different URL:

```jsx
<ConsoleHost
  module={myGame}
  connections={{ registryUrl: "https://my-mirror.example.com/index.json" }}
/>
```

As long as the JSON follows the same shape as `registry/index.json`, everything downstream just works. The default URL is a convenience; it is not a requirement.

## CLI

From the repo root:

| Command                                      | Effect                                                |
|----------------------------------------------|-------------------------------------------------------|
| `node scripts/validate-registry.mjs`          | Schema-check every manifest, report mistakes          |
| `node scripts/build-registry.mjs`             | Regenerate `registry/index.json`                      |
| `node scripts/build-registry.mjs --check`     | Exit non-zero if `index.json` is stale (CI uses this) |

None of them install any dependencies; they ship only with Node 20+.

## What does NOT belong here

- Arbitrary JavaScript. Themes are CSS + metadata. Projects are metadata + a URL to an external site. There is no code execution at registry time.
- Private or paywalled content. Everything here is public and linkable.
- Unlicensed artwork. Only ship `preview.png` / `cover.png` you own or have permission to use.

## Moderation

- Anyone can open a PR to hide (`status: "hidden"`) or remove an entry that turns out to be malicious, broken, or unmaintained. A maintainer will weigh the request.
- Listings are not endorsements. Inclusion in `projects/` means "this project exists and opts to be discoverable from other consoles," nothing more.
